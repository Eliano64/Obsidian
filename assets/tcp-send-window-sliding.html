/* 发送方收到 ACK=31, win=20 后 */
P1 = 31;  // 窗口后沿
P2 = 31;  // 尚未发送任何数据
P3 = 51;  // 窗口前沿 = 31+20

send_window  = P3 - P1 = 20;
inflight     = P2 - P1 = 0;
usable       = P3 - P2 = 20;

4. 三个指针的核心规则总结

P₁ 后沿只有两种可能：不动（无新确认）或前移（收到新累积确认）。
绝不后移——已确认的字节不可撤销，数据可从发送缓存删除。
P₂ 可用起点指向下一个允许发送但尚未发送的字节。发送数据时 P₂ 前移；收到 ACK 时 P₂ 不动。
P₃ 前沿三种可能：前移（常态·确认扩大窗口）、不动（无变化或变化抵消）、后缩（接收方缩小窗口——TCP 强烈不推荐）。
/* 三个指针的精确含义（用于编程实现） */
/* < P1 : 已确认可删除 */
/* P1 ~ P2 : 已发送未确认（inflight） */
/* P2 ~ P3 : 允许发送但未发送（usable） */
/* > P3 : 不允许发送 */

swnd    = P3 - P1;  // 发送窗口总大小
inflight = P2 - P1;  // 已发未确认
usable   = P3 - P2;  // 可用窗口/有效窗口
