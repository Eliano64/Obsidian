# 操作系统是什么

**操作系统**是控制和管理整个计算机系统硬件与软件资源的系统软件。它组织调度计算机工作，分配系统资源，并向用户和应用程序提供接口和运行环境。

从层次上看：

![1000](../assets/os-overview-layered-role.svg)

| 位置 | 作用 |
| --- | --- |
| 向下 | 管理 CPU、内存、磁盘、I/O 设备等硬件资源 |
| 向上 | 向用户和应用程序提供方便易用的接口 |
| 中间 | 把裸机扩展成更易用、更安全、更强的机器 |

操作系统的三个核心定位：

- **系统资源的管理者**：管理处理机、存储器、文件和设备。
- **向上层提供服务**：提供命令接口、程序接口、图形界面等。
- **最接近硬件的软件层**：直接处理时钟、中断、设备驱动、CPU 状态切换等底层机制。

> [!note] 扩充机器
> 裸机只提供硬件能力。操作系统把硬件能力组织起来，提供进程、文件、虚拟内存、设备抽象等机制，使用户看到的是一台更方便的“扩充机器”。

# 操作系统的发展历程

操作系统的演进始终围绕几组矛盾展开：人工操作跟不上机器速度、CPU 与 I/O 设备速度不匹配、系统吞吐量与用户交互相冲突，以及平均响应速度不能满足确定的时间约束。

![1000](../assets/os-development-timeline.svg)

## 单道批处理系统

早期计算机采用手工操作。程序员需要人工装入程序和数据，程序运行结束后再由操作员准备下一项任务。计算机在这些人工操作期间只能等待，机器时间被大量浪费。

**单道批处理系统**把若干作业成批提交，并由监督程序自动完成作业之间的衔接。内存中通常只保留一道用户作业；这道作业结束后，系统自动装入下一道作业。

- **改进**：减少人工装入和作业切换时间，使作业能够自动连续运行。
- **局限**：当前作业等待 I/O 时，CPU 也只能等待，因此处理机利用率仍然很低。
- **交互性**：用户提交作业后不能直接干预，只能等待整批作业处理完毕。

> [!note] “单道”统计的是驻留并运行的用户作业
> 监督程序仍然存在，但同一时段内存中只有一道用户作业。它的计算和 I/O 不能与另一道用户作业交错执行。

## 多道批处理系统

**多道批处理系统**允许多道作业同时进入内存。当正在运行的作业因 I/O 等待而阻塞时，操作系统把 CPU 分配给另一道就绪作业，使 CPU 和外设并行工作。

这需要考虑如下核心管理问题：

- 需要作业调度和进程调度决定谁进入内存、谁获得 CPU；
- 需要内存管理与存储保护，防止一道程序破坏另一道程序；
- 需要处理中断、资源分配以及多个程序之间的同步与互斥；
- 程序的执行顺序不再完全确定，形成并发和异步性。

它已有进程管理、内存管理等现代操作系统核心功能的雏形，显著提高了**资源利用率**和**系统吞吐量**。但用户仍然以脱机方式提交作业，无法在运行过程中及时交互，短作业也可能等待很久。

> [!important] 多道程序不等于多核并行
> 在单核 CPU 上，多道程序通过切换交替推进，属于并发；只有多个处理单元在同一时刻分别执行不同程序时才是并行。

## 分时操作系统

批处理系统关注单位时间完成多少作业，**分时操作系统**则更关注用户多久能够得到**响应**。系统把 CPU 时间划分为很短的**时间片**，轮流分配给多个终端用户的进程。只要切换足够快，每个用户都会感觉自己在独占计算机。

分时系统通常具有四个特征：

| 特征 | 含义 |
| --- | --- |
| 同时性 | 多个用户可以同时使用系统，底层通过快速切换并发执行 |
| 独立性 | 各用户感觉自己独立使用一台计算机，操作彼此隔离 |
| 及时性 | 用户请求能在较短、可接受的时间内得到响应 |
| 交互性 | 用户可以边输入、边观察结果、边调整下一步操作 |

分时系统建立在多道程序设计之上，但增加了抢占式调度、时间片和终端交互。它能够改善平均响应时间，却通常不承诺任务一定在某个截止时间前完成。

## 实时操作系统

**实时操作系统**用于需要在明确时间限制内响应外部事件的场景。它关注的不只是计算结果是否正确，还关注**结果是否在规定时间内产生**。

| 类型 | 时间约束 | 例子 |
| --- | --- | --- |
| 硬实时系统 | 错过截止时间会导致系统失败或严重后果 | 飞行控制、工业保护、部分医疗设备 |
| 软实时系统 | 偶尔超时会降低服务质量，但不会造成灾难性后果 | 音视频播放、在线通信、交互媒体 |

实时系统通常需要优先级调度、快速中断处理、可控的资源竞争和最坏响应时间分析。==实时不等于运算速度特别快==：它的核心是行为可预测，并能在规定的时间上界内完成关键任务。

## 网络、分布式与微机操作系统

- **网络操作系统**运行在相互连接但仍保持自治的计算机上，提供远程登录、文件共享、打印和通信等网络服务。用户通常知道资源位于另一台机器。
- **分布式操作系统**试图统一管理多台计算机的处理器、存储和设备，使用户尽量感觉自己面对的是一个整体系统。它比网络操作系统更强调位置透明、迁移透明和统一资源管理。
- **微机操作系统**面向个人计算机，经历了单用户单任务、单用户多任务，再到融合多用户、网络、安全和图形界面的通用系统。它描述的是机器规模和使用场景，不是一种独立的调度机制。

## 主要类型对比

| 系统类型 | 首要目标 | 关键机制 | 主要局限或约束 |
| --- | --- | --- | --- |
| 单道批处理 | 减少人工干预 | 作业成批提交、自动顺序执行 | I/O 时 CPU 空闲，无交互 |
| 多道批处理 | 提高利用率和吞吐量 | 多道程序、调度、内存保护 | 交互性差，管理复杂 |
| 分时系统 | 缩短交互响应时间 | 时间片、抢占、多终端 | 通常不保证截止时间 |
| 实时系统 | 在时间约束内可靠响应 | 优先级、截止时间和最坏响应分析 | 资源和调度约束严格 |

# 操作系统管理哪些资源

| 资源 | 管理内容 |
| --- | --- |
| 处理机 | 进程调度、CPU 分配、上下文切换 |
| 存储器 | 内存分配、地址转换、虚拟存储 |
| 文件 | 文件创建、删除、读写、目录、权限 |
| 设备 | 设备分配、I/O 控制、缓冲、驱动、中断处理 |

这些资源往往被多个进程共同使用，因此需要操作系统统一管理。否则，多个应用程序直接操作同一个硬件资源，容易导致混乱、数据破坏或安全问题。

# 用户接口

操作系统向上提供的接口可以分为三类。

| 接口     | 使用者      | 例子           | 特点                |
| ------ | -------- | ------------ | ----------------- |
| 联机命令接口 | 普通用户     | shell、cmd 命令 | 用户输入一条命令，系统执行一条命令 |
| 脱机命令接口 | 普通用户或管理员 | 批处理脚本        | 用户提交一组命令，系统按批执行   |
| 程序接口   | 程序员和应用程序 | 系统调用命令       | 程序通过代码请求内核服务      |

图形用户界面属于更直观的用户接口。它把许多命令操作包装成窗口、图标、菜单和按钮，但底层仍会转化为操作系统服务。

程序接口由一组 **system call（系统调用）** 组成。应用程序通过体系结构规定的入口指令请求内核服务，例如 x86-64 的 `syscall`；完整机制和入口指令不能混为一谈，详见 [[System-Call]]。状态切换见 [[CPU-Modes]]。

# 操作系统的基本特征

现代操作系统具有并发、共享、虚拟和异步四个基本特征。并发和共享是最基本的两个特征，虚拟和异步建立在二者之上，详见 [[OS-Characteristics]]。
